# Data Model: Save GraphNodes

**Feature**: 063-save-graphnodes
**Date**: 2026-04-01

## Entities

### GraphNodeTemplateMeta

Lightweight metadata extracted from a saved GraphNode `.flow.kts` file during discovery. Used for palette display without full deserialization.

| Field           | Type             | Required | Description                                             |
|-----------------|------------------|----------|---------------------------------------------------------|
| name            | String           | Yes      | Display name of the GraphNode template                  |
| description     | String           | No       | Optional description extracted from metadata header     |
| inputPortCount  | Int              | Yes      | Number of exposed input ports                           |
| outputPortCount | Int              | Yes      | Number of exposed output ports                          |
| childNodeCount  | Int              | Yes      | Number of child nodes in the composition                |
| filePath        | String           | Yes      | Absolute path to the `.flow.kts` template file          |
| tier            | PlacementLevel   | Yes      | Which tier the file was discovered at (MODULE/PROJECT/UNIVERSAL) |

**Identity**: Unique by `name` within a tier. Across tiers, deduplication keeps the most specific (lowest ordinal) tier.

**Source**: Extracted from `.flow.kts` file metadata comment header during filesystem scanning.

### GraphNode Template File Format

Each saved GraphNode is a `.flow.kts` file with a metadata comment header followed by the DSL representation.

```
/*
 * GraphNode Template: {Name}
 * @GraphNodeTemplate
 * @TemplateName {Name}
 * @Description {optional description}
 * @InputPorts {count}
 * @OutputPorts {count}
 * @ChildNodes {count}
 * Created: {timestamp}
 * License: Apache 2.0
 */

import io.codenode.fbpdsl.dsl.*
import io.codenode.fbpdsl.model.*

val template = graphNodeTemplate("{Name}") {
    // ... DSL content generated by FlowGraphSerializer
}
```

**Metadata markers** (parsed by regex during discovery):
- `@GraphNodeTemplate` — Presence marker distinguishing from regular .flow.kts files
- `@TemplateName {name}` — Template name
- `@Description {text}` — Optional description
- `@InputPorts {N}` — Input port count
- `@OutputPorts {N}` — Output port count
- `@ChildNodes {N}` — Child node count

### Existing Entities (Referenced, Not Modified)

#### GraphNode (fbpDsl model)
Already `@Serializable`. The saved template captures the full composition:
- `childNodes: List<Node>` — All child CodeNodes and nested GraphNodes
- `internalConnections: List<Connection>` — Wiring between children
- `portMappings: Map<String, PortMapping>` — External-to-internal port links
- `inputPorts / outputPorts: List<Port<*>>` — Exposed interface ports

#### PlacementLevel (graphEditor model)
Existing enum: MODULE, PROJECT, UNIVERSAL. Reused as-is for GraphNode tier selection.

#### NodeTypeDefinition (fbpDsl model)
Existing palette entry model for CodeNodes. GraphNodes use `GraphNodeTemplateMeta` instead — different structure since they represent compositions, not single-node definitions.

## Relationships

```
GraphNodeTemplateMeta (palette display)
    └── references → .flow.kts file (full template on disk)
         └── contains → GraphNode definition
              ├── childNodes: List<Node> (CodeNodes and/or nested GraphNodes)
              ├── internalConnections: List<Connection>
              └── portMappings: Map<String, PortMapping>

PlacementLevel
    └── determines → filesystem tier path for storage/discovery

GraphNodeTemplateRegistry
    └── holds → List<GraphNodeTemplateMeta> (merged from all tiers)
    └── provides → palette data + instantiation
```

## Validation Rules

1. **Name uniqueness**: A template name must be unique within a tier. Cross-tier duplicates are resolved by precedence (Module > Project > Universal).
2. **Non-empty name**: Template name must be non-blank.
3. **Port consistency**: Saved port mappings must reference valid child node IDs and port names.
4. **Level compatibility**: On save, all child CodeNode/IP type dependencies must be available at the target level (or promoted via the promotion dialog).
