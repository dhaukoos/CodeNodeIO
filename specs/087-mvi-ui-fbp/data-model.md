# Phase 1 Data Model: MVI Pattern for UI-FBP Interface Generation

**Revised 2026-05-03** — Design B (singleton eliminated; controller owns
all per-port flows).

## Generator-input model

The generator continues to consume `UIFBPSpec` (existing — see
`flowGraph-generate/src/commonMain/kotlin/io/codenode/flowgraphgenerate/parser/UIFBPSpec.kt`).
**No spec-side changes are required**; the existing `sinkInputs` and
`sourceOutputs` carry everything the new generators need.

```kotlin
data class UIFBPSpec(
    val flowGraphPrefix: String,           // drives all generated names ({Name}State, {Name}Event, …)
    val composableName: String,            // hand-written Screen function (consumes generated types)
    val viewModelTypeName: String,         // {Name}ViewModel
    val packageName: String,               // host module's base package
    val sourceOutputs: List<PortInfo>,     // → Event cases (one case per port)
    val sinkInputs: List<PortInfo>,        // → State properties (one prop per port)
    val ipTypeImports: List<String> = emptyList()
)

data class PortInfo(
    val name: String,
    val typeName: String,
    val isNullable: Boolean = false
)
```

## Generated artifact model (per UI-FBP module)

| File | Status | Role |
|------|--------|------|
| `viewmodel/{Name}State.kt` | **MODIFIED** (shape change) | Public immutable `data class` — UI's observable state snapshot. One `val` per `sinkInput`. |
| `viewmodel/{Name}Event.kt` | **NEW** | Public `sealed interface` — user intents the Screen can raise. One case per `sourceOutput`. |
| `viewmodel/{Name}ViewModel.kt` | **MODIFIED** (shape change) | Exposes `state: StateFlow<{Name}State>` + `fun onEvent({Name}Event)`. Forwarding control surface unchanged. |
| `controller/{Name}ControllerInterface.kt` | **MODIFIED** (additive only) | extends `ModuleController`; preserves per-sink-port `StateFlow<T>` members; ADDS `fun emit<SourcePort>(value: T)` per source port. |
| `controller/{Name}Runtime.kt` | **MODIFIED** (shape change) | Constructs per-port `MutableStateFlow` (sink) + `MutableSharedFlow` (source) instances; wraps Source/Sink CodeNodes via `withSource(...)` / `withReporter(...)` to inject those flows; returns `{Name}ControllerInterface` whose emit methods push to the source flows. |
| `nodes/{Name}SourceCodeNode.kt` | **MODIFIED** (shape change) | `createRuntime(name)` now produces a runtime that **collects from a controller-supplied `SharedFlow<T>` per source port** and emits to its FBP output channel. The wrapper `withSource(...)` returns a per-instance `CodeNodeDefinition` with the flows captured. |
| `nodes/{Name}SinkCodeNode.kt` | **MODIFIED** (shape change) | `createRuntime(name)` now produces a runtime that **calls a controller-supplied reporter callback per received IP** instead of writing to a singleton. The wrapper `withReporter(...)` returns a per-instance `CodeNodeDefinition` with the callback captured. |
| `userInterface/{Name}PreviewProvider.kt` (jvmMain) | unchanged | registers with PreviewRegistry. |

**No singleton emitted.** `{Name}State.kt` is the data class only; there is
no `{Name}StateStore.kt` companion. Today's `{Name}State.kt` (singleton
object) is wholesale replaced.

## Generated entities (semantic model)

### `{Name}State` (public, immutable)

```kotlin
package {basePackage}.viewmodel

data class {Name}State(
    val {sinkInput[0].name}: {sinkInput[0].kotlinType} = {default},
    val {sinkInput[1].name}: {sinkInput[1].kotlinType} = {default},
    // … one property per sinkInput
)
```

- One constructor `val` per `sinkInput`. Defaults follow the existing
  default-value table from `UIFBPStateGenerator`:
  - `Int` → `0`, `Long` → `0L`, `Double` → `0.0`, `Float` → `0.0f`,
    `Boolean` → `false`, `String` → `""`, nullable / unknown → `null`.
- Empty-state edge case: `data class {Name}State()` (zero-arg).
- Equality / hashCode / copy / toString are auto-generated by `data class`.

### `{Name}Event` (public, sealed)

```kotlin
package {basePackage}.viewmodel

sealed interface {Name}Event {
    // For each sourceOutput port carrying a non-Unit value:
    data class Update{PortName}(val value: {PortType}) : {Name}Event
    // For each sourceOutput port carrying Unit:
    data object {PortName} : {Name}Event
}
```

- One case per `sourceOutput`. Naming convention:
  - Valued port: `Update{PortName}` (PortName PascalCased) carrying the value.
  - `Unit`-typed port: `data object {PortName}` (no payload).
- Empty-events edge case: `sealed interface {Name}Event` with no cases.

### `{Name}ControllerInterface` (public, additive surface)

```kotlin
package {basePackage}.controller

interface {Name}ControllerInterface : ModuleController {
    // Sink-side (unchanged from today): one StateFlow<T> per sinkInput
    val {sinkPort}: StateFlow<{kotlinType}>

    // NEW (Design B): one emit method per sourceOutput
    fun emit{SourcePortName}(value: {kotlinType})
    // … or, for Unit-typed source ports:
    fun emit{SourcePortName}()
}
```

- Existing methods (`start / stop / pause / resume / reset / getStatus /
  executionState` from `ModuleController`, plus per-sink-port
  `StateFlow<T>`) are preserved unchanged (FR-007 additive).
- New `emit<SourcePort>` methods are the SOLE path from ViewModel to FBP
  graph for source-port emissions.

### `{Name}Runtime.kt` factory

```kotlin
package {basePackage}.controller

fun create{Name}Runtime(flowGraph: FlowGraph): {Name}ControllerInterface {
    // Per-flow-graph state — fresh on each call; no singletons.
    val _<sinkPort> = MutableStateFlow<T>({default})
    // … one MutableStateFlow per sinkInput
    val _<sourcePort> = MutableSharedFlow<T>(replay = 1, extraBufferCapacity = 64)
    // … one MutableSharedFlow per sourceOutput

    val controller = DynamicPipelineController(
        flowGraphProvider = { flowGraph },
        lookup = { name ->
            when (name) {
                "{Name}Source" -> {Name}SourceCodeNode.withSource(_<sourcePort>, …)
                "{Name}Sink"   -> {Name}SinkCodeNode.withReporter { value ->
                    _<sinkPort>.value = value as <T>
                }
                else -> null
            }
        },
        onReset = {
            _<sinkPort>.value = {default}
            // … reset every sink flow; source flows naturally drain
        }
    )

    return object : {Name}ControllerInterface, ModuleController by controller {
        override val <sinkPort>: StateFlow<<T>> = _<sinkPort>.asStateFlow()
        // … one override per sinkInput

        override fun emit<SourcePort>(value: <T>) {
            controller.coroutineScope.launch { _<sourcePort>.emit(value) }
        }
        // … one override per sourceOutput
    }
}
```

- Factory signature `create{Name}Runtime(flowGraph: FlowGraph): {Name}ControllerInterface`
  is unchanged — `ModuleSessionFactory` and the Runtime Preview panel
  continue to call it the same way.
- Per-flow-graph state lives in the closure of this function; multiple
  instances are state-isolated.
- `controller.coroutineScope` is the existing CoroutineScope on
  `DynamicPipelineController`; `emit<SourcePort>` uses it to dispatch
  source emissions without blocking the caller.

### `{Name}SinkCodeNode` (shape change)

```kotlin
package {basePackage}.nodes

object {Name}SinkCodeNode : CodeNodeDefinition {
    override val name = "{Name}Sink"
    override val category = CodeNodeType.SINK
    override val inputPorts = listOf(/* … one PortSpec per sinkInput */)
    override val outputPorts = emptyList<PortSpec>()

    override fun createRuntime(name: String): NodeRuntime =
        createSinkRuntime(name) { value -> /* default no-op */ }

    /** Returns a per-flow-graph wrapper that delivers received IPs to [reporter]. */
    fun withReporter(reporter: (Any?) -> Unit): CodeNodeDefinition = object : CodeNodeDefinition by this {
        override fun createRuntime(name: String): NodeRuntime =
            createSinkRuntime(name) { value -> reporter(value) }
    }

    private fun createSinkRuntime(name: String, onValue: (Any?) -> Unit): NodeRuntime { … }
}
```

- The bare `object` retains its `CodeNodeDefinition` identity (required by
  `NodeDefinitionRegistry` discovery + GraphEditor palette display).
- The default `createRuntime(name)` returns a runtime with a no-op reporter
  — useful when the SinkCodeNode is dropped onto a graph in the GraphEditor
  outside of a UI-FBP runtime context (no observers).
- `withReporter(reporter)` returns a delegated wrapper whose
  `createRuntime(name)` captures the per-flow-graph callback.

### `{Name}SourceCodeNode` (shape change)

```kotlin
package {basePackage}.nodes

object {Name}SourceCodeNode : CodeNodeDefinition {
    override val name = "{Name}Source"
    override val category = CodeNodeType.SOURCE
    override val inputPorts = emptyList<PortSpec>()
    override val outputPorts = listOf(/* … one PortSpec per sourceOutput */)

    override fun createRuntime(name: String): NodeRuntime =
        createSourceRuntime(name, /* per-port flows: empty / never-emit */)

    /** Returns a per-flow-graph wrapper that emits values from [sources] onto its output ports. */
    fun withSource(
        // one SharedFlow<T> per sourceOutput, in declared order
        source0: SharedFlow<T0>,
        source1: SharedFlow<T1>,
        // …
    ): CodeNodeDefinition = object : CodeNodeDefinition by this {
        override fun createRuntime(name: String): NodeRuntime =
            createSourceRuntime(name, source0, source1, /* … */)
    }

    private fun createSourceRuntime(name: String, /* sources */): NodeRuntime { … }
}
```

- Same singleton-`object` discipline as the SinkCodeNode.
- The default `createRuntime(name)` produces a runtime with never-emitting
  sources (useful in the GraphEditor palette).
- `withSource(...)` returns a delegated wrapper that captures the
  per-flow-graph source flows.

### `{Name}ViewModel` (public)

```kotlin
package {basePackage}.viewmodel

import androidx.lifecycle.ViewModel
import androidx.lifecycle.viewModelScope
import io.codenode.fbpdsl.model.ExecutionState
import io.codenode.fbpdsl.model.FlowGraph
import kotlinx.coroutines.flow.MutableStateFlow
import kotlinx.coroutines.flow.StateFlow
import kotlinx.coroutines.flow.asStateFlow
import kotlinx.coroutines.flow.update
import kotlinx.coroutines.launch
import {basePackage}.controller.{Name}ControllerInterface

class {Name}ViewModel(
    private val controller: {Name}ControllerInterface
) : ViewModel() {

    private val _state = MutableStateFlow({Name}State())
    val state: StateFlow<{Name}State> = _state.asStateFlow()

    val executionState: StateFlow<ExecutionState> = controller.executionState

    init {
        // For each sinkInput port: launch a collector that folds emissions into _state.
        viewModelScope.launch {
            controller.{sinkPort}.collect { value ->
                _state.update { it.copy({sinkPort} = value) }
            }
        }
        // … one launch per sinkInput
    }

    fun onEvent(event: {Name}Event) {
        when (event) {
            // For each Update{PortName}(value):
            is {Name}Event.Update{PortName} -> controller.emit{PortName}(event.value)
            // For each {PortName} data object (Unit-typed source port):
            {Name}Event.{PortName} -> controller.emit{PortName}()
        }
    }

    fun start(): FlowGraph = controller.start()
    fun stop(): FlowGraph = controller.stop()
    fun pause(): FlowGraph = controller.pause()
    fun resume(): FlowGraph = controller.resume()
    fun reset(): FlowGraph = controller.reset()
}
```

- Constructor unchanged (keeps `ModuleSessionFactory.tryCreateViewModel`
  reflection match working).
- `state` + `onEvent` are the new MVI surface.
- `onEvent` dispatches **through the controller** — no direct
  shared-singleton writes.
- Forwarding control surface (start/stop/pause/resume/reset) unchanged.
- The prior `emit(...)` aggregate is removed (FR-011).

## Cross-entity invariants

1. **One-to-one port↔generated-element mapping**:
   - every `sinkInput` produces exactly one State property AND one
     Controller `StateFlow<T>` AND one Runtime `MutableStateFlow<T>` AND
     one ViewModel `init` collector.
   - every `sourceOutput` produces exactly one Event case AND one
     Controller `emit<Port>` method AND one Runtime `MutableSharedFlow<T>`
     AND one ViewModel `onEvent` branch AND one SourceCodeNode
     `withSource(...)` parameter.
2. **No singletons hold per-flow-graph state.** The only singletons in the
   generated artifact set are the `CodeNodeDefinition` `object`s themselves
   (Source/Sink) — and these hold no per-flow-graph state; their per-instance
   wrappers (`withReporter`/`withSource`) do.
3. **State data class is the SSOT for the UI.** The ViewModel's `_state`
   is the only authority for what the screen renders. The controller's
   per-port flows are the authority for the runtime side. The two sides
   communicate only through the controller's typed observables.
4. **Test isolation by construction.** Two `create{Name}Runtime(flowGraph)`
   calls return two independent controllers with independent flows;
   constructing two `{Name}ViewModel` instances thus produces two
   independent state streams.
5. **Determinism**: regeneration on an unchanged spec produces
   byte-identical output (SC-005). Iterate over `sinkInputs` /
   `sourceOutputs` in their declared order; no `Map`-based intermediates.
