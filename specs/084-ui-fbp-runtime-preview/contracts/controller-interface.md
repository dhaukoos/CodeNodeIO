# Contract: `{FlowGraph}ControllerInterface` (post-085)

This contract defines the shape that every UI-FBP-generated `{FlowGraph}ControllerInterface` MUST satisfy. The interface is implemented by **two different concrete types** depending on the runtime path:

1. **Runtime Preview path** (in-process, GraphEditor) — implemented by a `java.lang.reflect.Proxy` constructed by `flowGraph-execute/jvmMain/.../ModuleSessionFactory.createControllerProxy`. Method calls reflect to `DynamicPipelineController` (inherited `ModuleController` surface) and `{FlowGraph}State.{port}Flow` fields (state-flow getters).
2. **Production-app deployment path** — implemented by the anonymous object expression returned from `create{FlowGraph}Runtime(flowGraph)`, which is emitted by `ModuleRuntimeGenerator`:

   ```kotlin
   fun create{FlowGraph}Runtime(flowGraph: FlowGraph): {FlowGraph}ControllerInterface {
       val controller = DynamicPipelineController(
           flowGraphProvider = { flowGraph },
           lookup = {FlowGraph}NodeRegistry::lookup,
           onReset = {FlowGraph}State::reset
       )
       return object : {FlowGraph}ControllerInterface, ModuleController by controller {
           override val y = {FlowGraph}State.yFlow
           // ...
       }
   }
   ```

The interface SHAPE must satisfy both: every member must be implementable by both the proxy AND by the Runtime factory's anonymous object. Both reference shapes constrain what the generator may emit.

## Sources of truth (post-085)

- **Runtime Preview path**: `flowGraph-execute/src/jvmMain/kotlin/io/codenode/flowgraphexecute/ModuleSessionFactory.kt` — the `createControllerProxy` method's `InvocationHandler`, including the explicit `"getStatus" -> controller.getStatus()` case added by feature 085.
- **Production-app path**: `CodeNodeIO-DemoProject/StopWatch/src/commonMain/kotlin/io/codenode/stopwatch/controller/StopWatchRuntime.kt` (canonical reference shape, generated by `ModuleRuntimeGenerator`); `flowGraph-generate/src/commonMain/kotlin/io/codenode/flowgraphgenerate/generator/ModuleRuntimeGenerator.kt` (the generator itself).
- **Inherited control surface**: `fbpDsl/src/commonMain/kotlin/io/codenode/fbpdsl/runtime/ModuleController.kt` (interface, with default `getStatus()` impl); `fbpDsl/src/commonMain/kotlin/io/codenode/fbpdsl/runtime/DynamicPipelineController.kt` (production implementer of the inherited surface).

## Inheritance

The interface MUST extend `io.codenode.fbpdsl.runtime.ModuleController`:

```kotlin
package io.codenode.{module}.controller

import io.codenode.fbpdsl.runtime.ModuleController
import kotlinx.coroutines.flow.StateFlow
// + IP type imports per spec.ipTypeImports

interface {FlowGraph}ControllerInterface : ModuleController {
    // sink-input flow members ONLY (see "Required members" below)
}
```

Generators MUST NOT redeclare any member that `ModuleController` already declares. Specifically, the following come from inheritance and MUST NOT appear on `{FlowGraph}ControllerInterface`:

- `start(): FlowGraph`, `stop(): FlowGraph`, `pause(): FlowGraph`, `resume(): FlowGraph`, `reset(): FlowGraph`
- `getStatus(): FlowExecutionStatus` (default impl in `ModuleController`; override in `DynamicPipelineController`)
- `setAttenuationDelay(Long)`, `setEmissionObserver(EmissionObserver?)`, `setValueObserver(ValueObserver?)`
- `executionState: StateFlow<ExecutionState>`

## FQCN resolution

The Runtime Preview path's `ModuleSessionFactory` expects to find the interface class at exactly one of these fully-qualified names (resolved in order; first match wins):

| Order | FQCN | Status |
|---|---|---|
| 1 | `io.codenode.{module-lowercase}.controller.{FlowGraph}ControllerInterface` | **TARGET** for this feature |
| 2 | `io.codenode.{module-lowercase}.generated.{FlowGraph}ControllerInterface` | Legacy fallback — UI-FBP MUST NOT emit here |

`{module-lowercase}` is the lowercase form of the module's package name suffix. `{FlowGraph}` is the flow graph's prefix name from the user-selected `.flow.kt` file (per the post-082/083 naming model, NOT the module's name).

## Required members

### Inherited from `ModuleController` — MUST NOT be redeclared

See "Inheritance" above. Both runtime paths satisfy these members:

| Member | Runtime Preview path | Production-app path |
|---|---|---|
| `start/stop/pause/resume/reset` | `controller.{same}(); flowGraphProvider()` (proxy at `ModuleSessionFactory.createControllerProxy`) | `DynamicPipelineController.{same}()` via `ModuleController by controller` delegation |
| `getStatus` | `controller.getStatus()` (explicit case in proxy added by feature 085) | `DynamicPipelineController.getStatus()` override |
| `setAttenuationDelay`, `setEmissionObserver`, `setValueObserver` | proxy delegates to `controller.{same}` | `ModuleController by controller` delegation |
| `executionState: StateFlow<ExecutionState>` | proxy returns `controller.executionState` directly | `ModuleController by controller` delegation |

### Sink-input state-flow members — UI-FBP-specific (variable count, derived from spec)

For each `y` in `UIFBPSpec.sinkInputs` (display outputs the UI observes), the interface MUST declare:

```kotlin
val y: StateFlow<T>
```

`UIFBPSpec.sourceOutputs` are NOT mirrored on the interface — the UI emits them via `viewModel.emit(...)` writing to `{FlowGraph}State._x` mutable fields, but does not observe them back through the controller. This matches `AddressesControllerInterface` and `WeatherForecastControllerInterface` (entity-module precedent).

**Runtime Preview path** — proxy resolution: the `InvocationHandler` intercepts the synthetic getter `getY()` (from Kotlin's `val y`), strips the `get` prefix, lower-cases the first char, and reflects:

1. `stateObject.javaClass.getField("${propName}Flow")` — if found, return its value.
2. Else `stateObject.javaClass.getMethod("get${propName.capitalize()}Flow")` — if found, invoke and return.
3. Else return `null` (downstream `StateFlow<T>` typing surfaces as NPE).

**Production-app path** — the anonymous object expression in `create{FlowGraph}Runtime(...)` writes `override val y = {FlowGraph}State.yFlow` for each sink-input port. `{FlowGraph}State` exposes the read-only flows (Kotlin's standard `val yFlow: StateFlow<T> = _y.asStateFlow()` pattern).

**Implication for the generator**: for every `val y` the UI-FBP generator adds to the interface, `{FlowGraph}State` MUST expose a public field `yFlow`. UI-FBP's `UIFBPStateGenerator` already meets this — the generated State carries `_y: MutableStateFlow<T>` plus `val yFlow: StateFlow<T> = _y.asStateFlow()`.

### Disallowed members

- No suspend functions (proxy's `InvocationHandler` is non-suspending; production-app `ModuleController by controller` delegation also doesn't suspend).
- No methods with parameters other than the inherited control members (which take none on the interface — the parameter-bearing ones are inherited from `ModuleController` and are not redeclared by the UI-FBP generator).
- No generic type parameters on the interface itself.
- No re-declaration of `ModuleController` members (see "Inheritance" — all such members come from inheritance).

## State object companion contract

In addition to the interface contract above, the module's `{FlowGraph}State` Kotlin `object` MUST satisfy:

| Requirement | Why |
|---|---|
| Be a Kotlin `object` (single `INSTANCE` field) | Runtime Preview: `tryGetStateObject` reads `stateClass.getField("INSTANCE").get(null)`. Production app: the Runtime factory's `onReset = {FlowGraph}State::reset` and the anonymous `override val y = {FlowGraph}State.yFlow` references rely on the singleton shape. |
| Live at `io.codenode.{module}.viewmodel.{FlowGraph}State` (preferred) or `io.codenode.{module}.{FlowGraph}State` (fallback) | Runtime Preview lookup order; production-app generator emits imports against the canonical `viewmodel/` location. |
| Expose `{y}Flow: StateFlow<T>` field for every `val {y}: StateFlow<T>` in the interface | Both paths read state via these fields. |
| Expose internal `_{y}: MutableStateFlow<T>` companions | Source CodeNode runtimes and `viewModel.emit(...)` write to these. |
| Expose a `fun reset()` method | Runtime Preview: `createResetCallback` references it. Production app: `onReset = {FlowGraph}State::reset` is passed into `DynamicPipelineController`. |

UI-FBP's `UIFBPStateGenerator` already meets all five requirements; the only changes for this feature are (a) the `viewmodel/` subpackage location (Decision 4) and (b) the `{FlowGraph}` prefix derivation (Decision 2; see research).

## ViewModel companion contract

| Requirement | Why |
|---|---|
| Live at `io.codenode.{module}.viewmodel.{FlowGraph}ViewModel` (preferred) or `io.codenode.{module}.{FlowGraph}ViewModel` (fallback) | Runtime Preview lookup; production app imports against the canonical location. |
| Have a public single-arg constructor whose parameter type is assignable from the `ControllerInterface` (UI-FBP) or a two-arg `(ControllerInterface, Dao)` (entity modules with persistence) | Runtime Preview: `tryCreateViewModel`. Production app: the consumer constructs `{FlowGraph}ViewModel(controller)` (or with a DAO when applicable). |
| Extend `androidx.lifecycle.ViewModel` (multiplatform variant via `lifecycle-viewmodel-compose`) | Required by GraphEditor's preview hosting AND by the production app's Compose lifecycle. |
| Expose state flows by reading `{FlowGraph}State.{y}Flow` directly (not via the controller) | Matches WeatherForecast/Addresses precedent; same code works under both runtime paths because both mutate the same State singleton. |

## PreviewProvider companion contract

The UI-FBP-generated `{FlowGraph}PreviewProvider.kt` (in `jvmMain/.../userInterface/`) MUST:

| Requirement | Why |
|---|---|
| Live at `io.codenode.{module}.userInterface.{FlowGraph}PreviewProvider` (jvmMain source set) | `DynamicPreviewDiscovery` scans `userInterface/` for files ending in `PreviewProvider.kt`. |
| Be a Kotlin `object` with a `fun register()` method | Discovery reflectively reads the `INSTANCE` field and invokes `register()`. |
| Call `PreviewRegistry.register("{flowGraphPrefix}") { viewModel, modifier -> ... }` | The registry key MUST equal the flow graph prefix because `RuntimePreviewPanel` looks up by `flowGraphName`. |
| Inside the lambda, cast `viewModel as {FlowGraph}ViewModel` and invoke the user-authored Composable: `{composableName}(viewModel = vm, modifier = modifier)` | Per Decision 2: the registry key (flow graph prefix) and Composable function call (user-authored composable name) are potentially-distinct identifiers; `PreviewProviderGenerator` (extended for this feature) takes both as inputs. |

## Failure modes

| Failure | Symptom — Runtime Preview path | Symptom — production app path |
|---|---|---|
| `ControllerInterface` class not loadable (FQCN wrong, module not on classpath) | `createSession` returns a session with `Any()` ViewModel; PreviewProvider's cast throws `ClassCastException`. | Compile-time error in the app (the import doesn't resolve). |
| `ControllerInterface` loadable but doesn't extend `ModuleController` | `ModuleController by controller` delegation in the Runtime factory fails to compile | Compile-time error in the generated `{FlowGraph}Runtime.kt`. |
| `ControllerInterface` loadable but `ViewModel` constructor doesn't match | ViewModel is `Any()`; cast in PreviewProvider throws. | Compile-time error in the app's DI wiring. |
| `{FlowGraph}State.{y}Flow` field missing for an interface getter | Proxy returns `null`; ViewModel reads `null` from State directly; UI `collectAsState()` throws NPE. | The Runtime factory's `override val y = {FlowGraph}State.yFlow` won't compile (missing field). |
| `{FlowGraph}State` not a Kotlin `object` (no `INSTANCE` field) | `tryGetStateObject` returns null; reset callback is null; state-flow access via reflection fails. | The Runtime factory's `onReset = {FlowGraph}State::reset` won't compile (object reference required). |
| `{FlowGraph}NodeRegistry` lookup returns null for a node name in the FlowGraph | `DynamicPipelineController` start fails at runtime with an unresolved-node error | Same — both paths use `DynamicPipelineController`. |
| `{FlowGraph}PreviewProvider` not loadable or `register()` throws | `PreviewRegistry.get(name)` returns null; panel shows "Preview not available for: {name}". | N/A (production app does not use `PreviewProvider`). |
| `{FlowGraph}PreviewProvider` registers under a key that doesn't match the user-selected flow graph name | Panel shows "Preview not available" even though the file exists | N/A. |
| `{FlowGraph}PreviewProvider`'s lambda calls a Composable function name that doesn't exist in the `userInterface/` package | Compile-time error in the generated PreviewProvider | N/A. |

Generators MUST emit code that satisfies every requirement above; the test plan (`tasks.md` once generated) MUST include reflection-based tests for the Runtime Preview contract AND compile-time tests (build the generated module under JVM target) for the production-app contract.

## Versioning

This contract is internal to the project — there is no external consumer to version against. Changes to either runtime path (`ModuleSessionFactory`'s reflection rules OR `ModuleRuntimeGenerator`'s emission shape OR `ModuleController`'s default-method set) MUST be paired with corresponding generator updates in this feature's generator surface. The `Generated by CodeNodeIO {Generator}` marker comments in each emitted file give a static cue for any future global migration. Feature 085 already exercised this discipline by extending `ModuleController` with `getStatus()` and updating the proxy in lockstep with the generator emission shape; UI-FBP modules inherit the same surface.
