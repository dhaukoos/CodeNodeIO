# Contract: `{Module}ControllerInterface`

> **POST-085 INTEGRATION NOTE (2026-04-28)**: Feature 085 (universal-runtime collapse) shipped 2026-04-28 and changed both implementations of this contract. The interface itself now **extends `ModuleController`** (`interface X : ModuleController`); the `start/stop/pause/resume/reset/getStatus/setAttenuationDelay/setEmissionObserver/setValueObserver` methods inherit from `ModuleController` and MUST NOT be redeclared. The state-flow members are now `override val y: StateFlow<T>` (not freestanding `val`). The "Production-app deployment path" implementation is no longer a per-module `{Module}Controller` class — that class is **deleted**. It's replaced by the anonymous object inside the post-085 `create{Module}Runtime(flowGraph)` factory: `object : {Module}ControllerInterface, ModuleController by controller { override val y = {Module}State.yFlow ... }`. The "Sources of truth" for the production-app shape are now `controller/{Module}Runtime.kt` (e.g., `CodeNodeIO-DemoProject/StopWatch/.../controller/StopWatchRuntime.kt`) and `flowGraph-generate/.../generator/ModuleRuntimeGenerator.kt`. The Runtime Preview proxy path survives — `ModuleSessionFactory.createControllerProxy` was updated with explicit `getStatus` handling. The "Required members" / "Disallowed members" tables below need correspondingly tightened wording: control methods come from inheritance; state-flow getters are the only members the UI-FBP generator adds. See [CROSS-CHECK-085.md](../CROSS-CHECK-085.md) for the line-by-line delta.

This contract defines the shape that every UI-FBP-generated `{Module}ControllerInterface` MUST satisfy. The interface is implemented by **two different concrete types** depending on the runtime path:

1. **Runtime Preview path** (in-process, GraphEditor) — implemented by a `java.lang.reflect.Proxy` constructed by `flowGraph-execute/ModuleSessionFactory.createControllerProxy`. Method calls reflect to `DynamicPipelineController` (control methods) and `{Module}State.{port}Flow` fields (state-flow getters).
2. **Production-app deployment path** — implemented directly by the generated `{Module}Controller` class (emitted by `RuntimeControllerGenerator`). Method calls go through the controller's hand-coded `{Module}Flow()` runtime instance. **(POST-085: replaced by the anonymous `object` inside `create{Module}Runtime(flowGraph)` — see banner above.)**

The interface SHAPE must satisfy both: every member must be implementable by both the proxy AND by `{Module}Controller`. Both reference shapes (proxy resolution rules + the `{Module}Controller` class structure) constrain what the generator may emit.

## Sources of truth

- **Runtime Preview path**: `flowGraph-execute/src/jvmMain/kotlin/io/codenode/flowgraphexecute/ModuleSessionFactory.kt:108-150` (proxy handler).
- **Production-app path**: `CodeNodeIO-DemoProject/Addresses/src/commonMain/kotlin/io/codenode/addresses/controller/AddressesController.kt` (reference shape generated by `RuntimeControllerGenerator`).

## FQCN resolution

The proxy expects to find the interface class at exactly one of these fully-qualified names (resolved in order; first match wins):

| Order | FQCN | Status |
|---|---|---|
| 1 | `io.codenode.{moduleName.lowercase()}.controller.{ModuleName}ControllerInterface` | **TARGET** for this feature |
| 2 | `io.codenode.{moduleName.lowercase()}.generated.{ModuleName}ControllerInterface` | Legacy fallback — UI-FBP MUST NOT emit here |

`{moduleName.lowercase()}` is the lowercase form of `UIFBPSpec.moduleName`. `{ModuleName}` is the original casing.

## Required members

### Control methods

For every generated interface, the following five methods MUST be present with these names and return `io.codenode.fbpdsl.model.FlowGraph`:

```kotlin
fun start(): FlowGraph
fun stop(): FlowGraph
fun pause(): FlowGraph
fun resume(): FlowGraph
fun reset(): FlowGraph
```

`FlowGraph` is required (not `Unit`) because `{Module}Controller` (production-app path) implements these via `controller.startAll()` etc., which return `FlowGraph`, and `RuntimeControllerGenerator` emits the `FlowGraph` return type unconditionally. The Runtime Preview proxy also returns `FlowGraph` (via `flowGraphProvider()`), so both paths align.

| Interface method | Proxy behavior (Runtime Preview) | `{Module}Controller` behavior (production app) |
|---|---|---|
| `start` | `controller.start(); flowGraphProvider()` | `controller.startAll()`; sets `executionState = RUNNING`; launches `flow.start(scope)` |
| `stop` | `controller.stop(); flowGraphProvider()` | `controller.stopAll()`; calls `flow.stop()`; cancels scope |
| `pause` | `controller.pause(); flowGraphProvider()` | `controller.pauseAll()`; sets `executionState = PAUSED` |
| `resume` | `controller.resume(); flowGraphProvider()` | `controller.resumeAll()`; sets `executionState = RUNNING` |
| `reset` | `controller.reset(); flowGraphProvider()` | calls `stop()` then `flow.reset()` |

### Execution-state property

```kotlin
val executionState: StateFlow<ExecutionState>
```

The Kotlin synthetic getter is `getExecutionState()`. The proxy returns `controller.executionState` directly — a `StateFlow<ExecutionState>` already cold-publishing the dynamic pipeline's state.

### State-flow getter properties (variable count, derived from the spec)

For each `y` in `UIFBPSpec.sinkInputs` (display outputs the UI observes), the interface MUST declare:

```kotlin
val y: StateFlow<T>
```

`UIFBPSpec.sourceOutputs` are NOT mirrored on the interface — the UI emits them via `viewModel.emit(...)` but does not observe them back through the controller. This matches `AddressesControllerInterface` and `WeatherForecastControllerInterface`.

**Runtime Preview path** — proxy resolution (line 124-141 of `ModuleSessionFactory.kt`):

1. `propName = methodName.removePrefix("get").replaceFirstChar { it.lowercase() }`
2. Lookup `stateObject.javaClass.getField("${propName}Flow")` — if found, return its value.
3. Else `stateObject.javaClass.getMethod("get${propName.capitalize()}Flow")` — if found, invoke and return.
4. Else return `null` (which the caller's `StateFlow<T>` typing will coerce to a NPE downstream).

**Production-app path** — `{Module}Controller` implements each `val y: StateFlow<T>` as a member backed by its `{Module}Flow()` runtime instance (e.g., `val save: StateFlow<Any?> = flow.saveFlow` in `AddressesController.kt:61-65`). The same `{Module}State.{y}Flow` field is the ultimate source on both paths because `{Module}Flow`'s sink runtime writes into `{Module}State._{y}` and the State exposes the read-only flow.

**Implication for the generator**: For every `val y` in the interface, `{Module}State` MUST expose a public field `yFlow` (Kotlin's `val yFlow: StateFlow<T> = _y.asStateFlow()` already produces this). Today's UI-FBP `State` generator already meets this — verified in `TestModule/src/commonMain/kotlin/io/codenode/demo/viewmodel/DemoUIState.kt`.

### Disallowed members

- No suspend functions (proxy's `InvocationHandler` is non-suspending; production-app `{Module}Controller` doesn't emit suspend overrides either).
- No methods with parameters other than the listed control methods (which take none).
- No generic type parameters on the interface itself.

## State object companion contract

In addition to the interface contract above, the module's `{Module}State` Kotlin `object` MUST satisfy:

| Requirement | Why |
|---|---|
| Be a Kotlin `object` (single `INSTANCE` field) | Runtime Preview: `tryGetStateObject` reads `stateClass.getField("INSTANCE").get(null)` (line 207-208). Production app: `{Module}Controller`/`{Module}Flow` reference State as an object singleton (e.g., `AddressesState._save.value = …`). |
| Live at `io.codenode.{modulename}.viewmodel.{Module}State` (preferred) or `io.codenode.{modulename}.{Module}State` (fallback) | Runtime Preview lookup order at line 204-205; production-app generators emit imports against the canonical `viewmodel/` location. |
| Expose `{y}Flow: StateFlow<T>` field for every `val {y}: StateFlow<T>` in the interface | Both paths read state via these fields (proxy by reflection; controller by direct member access). |
| Expose internal `_{y}: MutableStateFlow<T>` companions | Source CodeNode runtimes and `viewModel.emit(...)` write to these; sink CodeNode runtimes write to them as well via `{Module}Flow`. |
| Expose a `fun reset()` method | Runtime Preview: `createResetCallback` at line 191-201. Production app: `{Module}Controller.reset()` calls `flow.reset()` which in turn calls `State.reset()`. |

UI-FBP's existing `UIFBPStateGenerator` already meets all five requirements; the only change for this feature is the package location (`viewmodel/` instead of base package — see plan / research Decision 6).

## ViewModel companion contract

| Requirement | Why |
|---|---|
| Live at `io.codenode.{modulename}.viewmodel.{Module}ViewModel` (preferred) or `io.codenode.{modulename}.{Module}ViewModel` (fallback) | Runtime Preview lookup at line 96-98. Production app imports against the canonical location. |
| Have a public single-arg constructor whose parameter type is assignable from the `ControllerInterface` (UI-FBP) or a two-arg `(ControllerInterface, Dao)` (entity modules) | Runtime Preview: `tryCreateViewModel` at line 162-188. Production app: the app's DI container constructs the ViewModel with a real `{Module}Controller` instance. |
| Extend `androidx.lifecycle.ViewModel` (multiplatform variant via `lifecycle-viewmodel-compose`) | Required by GraphEditor's preview hosting AND by the production app's Compose lifecycle. |
| Expose state flows by reading `{Module}State.{y}Flow` directly (not via the controller) | Matches WeatherForecast/Addresses precedent; same code works under both runtime paths because both mutate the same State singleton. |

## Failure modes

| Failure | Symptom — Runtime Preview path | Symptom — production app path |
|---|---|---|
| `ControllerInterface` class not loadable (FQCN wrong, module not on classpath) | `createSession` returns a session with `Any()` ViewModel; PreviewProvider's cast throws `ClassCastException`. | Compile-time error in the app (the import doesn't resolve). |
| `ControllerInterface` loadable but `ViewModel` constructor doesn't match | ViewModel is `Any()`; cast in PreviewProvider throws. | Compile-time error in the app's DI wiring. |
| `{Module}State.{y}Flow` field missing for an interface getter | Proxy returns `null`; ViewModel reads `null` from State directly (since this feature reads State, not controller); UI `collectAsState()` throws NPE. | `{Module}Controller` won't compile (its `val y: StateFlow<T> = flow.yFlow` member references the missing field). |
| `{Module}State` not a Kotlin `object` (no `INSTANCE` field) | `tryGetStateObject` returns null; reset callback is null; state-flow access via reflection fails. | `{Module}Controller` and `{Module}Flow` won't compile (they reference State as an object singleton). |
| `{Module}Controller` constructor doesn't match what the production app's DI expects | N/A | Compile-time or DI-init error in the app. |
| `{Module}Flow` won't compile (e.g., references a renamed CodeNode runtime class) | N/A (Runtime Preview uses dynamic pipeline, not `{Module}Flow`) | Module won't compile at all; deployable artifact is invalid. |
| `{Module}PreviewProvider` not loadable or `register()` throws | `PreviewRegistry.get(name)` returns null; panel shows "Preview not available for: {name}". | N/A (production app does not use `PreviewProvider`). |

Generators MUST emit code that satisfies every requirement above; the test plan (`tasks.md` once generated) MUST include reflection-based tests for the Runtime Preview contract AND compile-time tests (build the generated module under JVM target) for the production-app contract.

## Versioning

This contract is internal to the project — there is no external consumer to version against. Changes to either runtime path (`ModuleSessionFactory`'s reflection rules OR the entity-module generator emission shape) MUST be paired with corresponding generator updates in this feature's generator surface. The "Generated by CodeNodeIO" marker comments in each emitted file give a static cue for any future global migration — including the planned follow-up that collapses the thick stack onto a universal `DynamicPipelineController`-based runtime in fbpDsl (per spec Clarifications Q2).
