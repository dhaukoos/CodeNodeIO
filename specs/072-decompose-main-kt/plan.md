**Branch**: `072-decompose-main-kt` | **Date**: 2026-04-13 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/072-decompose-main-kt/spec.md`

## Summary

Decompose the 1923-line `Main.kt` in the graphEditor module into smaller, focused files. The `GraphEditorApp` composable (~1415 lines) is broken into logical sub-composables and extracted to separate files. Utility functions, `TopToolbar`, and `StatusBar` are moved to their own files. The final `Main.kt` contains only the `main()` function and application window setup.

### Phase 3: Final Main.kt Reduction (US3)

Move `GraphEditorApp` composable to `ui/GraphEditorApp.kt`. Main.kt retains only:
- `main()` function (lines 1886–1923)
- Koin initialization
- Window creation calling `GraphEditorApp()`

### State Sharing Strategy

The `GraphEditorApp` decomposition requires passing state between sub-composables. Two approaches:

1. **Parameter passing** (preferred for small interfaces): Sub-composables receive only what they need
2. **State holder class** (for initialization): A `GraphEditorState` data class bundles the 30+ state variables from the init section, returned by a `rememberGraphEditorState()` composable

We use approach 2 for initialization state and approach 1 for layout sub-composables, keeping each sub-composable's parameter list focused.
