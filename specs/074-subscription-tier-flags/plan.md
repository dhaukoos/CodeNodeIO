# Implementation Plan: Subscription Tier Feature Flags

**Branch**: `074-subscription-tier-flags` | **Date**: 2026-04-17 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/074-subscription-tier-flags/spec.md`

## Summary

Monetize CodeNodeIO as a SaaS desktop application with three subscription tiers (Free, Pro, Sim). This requires: (1) separating Save from code generation so Save writes only the `.flow.kt` file while a new "Generate Module" action handles full module scaffolding, (2) introducing a `SubscriptionTier` enum and `FeatureGate` service in `fbpDsl` that holds the current tier as a reactive `StateFlow`, (3) gating code generation behind Pro tier and Runtime Preview behind Sim tier, and (4) filtering the node and IP type palettes by tier.

## Save/Generate Separation Design

### Current Flow (monolithic)
```
Save button → ModuleSaveService.saveModule() → writes .flow.kt + generates everything
```

### New Flow (separated)
```
Save button → FlowGraphSerializer.serialize() → writes .flow.kt only
Generate button → ModuleSaveService.saveModule() → generates full module (incl. .flow.kt)
```

### Key Change in ModuleSaveService
Extract the `.flow.kt`-only write into a standalone method:
```
saveFlowKtOnly(flowGraph, outputDir, portTypeNames) → writes only the .flow.kt file
saveModule(flowGraph, outputDir, ...) → full module generation (calls saveFlowKtOnly internally)
```

### Save Location Tracking
Currently, `lastSaveDir` tracks where the user last saved. After separation:
- Save sets `lastSaveDir` (the directory containing the `.flow.kt`)
- Generate uses `lastSaveDir` as default output, or prompts if not set
- Both actions share the same directory context

## Feature Gate Design

### SubscriptionTier Enum (fbpDsl/commonMain)
```kotlin
enum class SubscriptionTier {
    FREE, PRO, SIM;
    
    fun hasAccess(required: SubscriptionTier): Boolean = this.ordinal >= required.ordinal
}
```

### FeatureGate Interface (fbpDsl/commonMain)
```kotlin
interface FeatureGate {
    val currentTier: StateFlow<SubscriptionTier>
    
    fun canGenerate(): Boolean = currentTier.value.hasAccess(SubscriptionTier.PRO)
    fun canSimulate(): Boolean = currentTier.value.hasAccess(SubscriptionTier.SIM)
    fun canUseRepositoryNodes(): Boolean = currentTier.value.hasAccess(SubscriptionTier.PRO)
}
```

### LocalFeatureGate (fbpDsl/jvmMain)
Reads `~/.codenode/config.properties` key `subscription.tier`. Defaults to `FREE`. Exposes `setTier()` for runtime updates that write back to the file and update the `StateFlow`.

## Palette Filtering Design

### Node Palette
- Free tier: All standard node types (all arities) + all custom (user-defined) nodes + all IP types
- Pro tier adds: Entity/repository nodes (NodeTypeDefinition category = "Repository") and future Pro-only categories
- Filter point: `NodePalette.kt` — check `featureGate.canUseRepositoryNodes()` before including Repository-category nodes

### IP Type Palette
- No filtering needed — all IP types are available at all tiers

## Runtime Preview Gating Design

Gate at `RuntimePreviewPanel.kt` content level:
- When `featureGate.canSimulate()` is false: Replace panel content with an upgrade prompt
- When true: Show full preview controls (existing behavior)
- The `CollapsiblePanel` wrapper remains functional regardless of tier
