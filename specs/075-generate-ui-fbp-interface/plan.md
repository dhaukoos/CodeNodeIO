# Implementation Plan: Generate UI-FBP Interface

**Branch**: `075-generate-ui-fbp-interface` | **Date**: 2026-04-19 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/075-generate-ui-fbp-interface/spec.md`

## Summary

Create a "Generate UI-FBP" function that analyzes a Compose UI file and generates four interface files bridging the UI to a flow-based processing graph: a ViewModel, a State object, a CodeNode Source definition, and a CodeNode Sink definition. The generator parses the UI file's ViewModel usage patterns ‚Äî `viewModel.emit(...)` calls become Source output ports, and `viewModel.{prop}.collectAsState()` calls become Sink input ports. The generated files follow the patterns established by existing modules (EdgeArtFilter, WeatherForecast, StopWatch) and the DemoUI prototype in the TestModule.

## UI File Parsing Design

### Input: A Compose UI `.kt` file

The parser extracts the ViewModel interface from usage patterns in the UI file:

### Step 1: Extract Module Name and ViewModel Type
```
@Composable fun {Name}(viewModel: {Name}ViewModel, ...) ‚Üí name = "{Name}", vmType = "{Name}ViewModel"
```
Regex: `fun\s+(\w+)\s*\(\s*viewModel\s*:\s*(\w+)`

### Step 2: Extract Source Outputs (emit function calls)
```
viewModel.emit(a, b)  ‚Üí find declaration: fun emit(numA: Double, numB: Double)
```
Regex on UI file: `viewModel\.(\w+)\(` (excluding collectAsState, collectAsState)
Then infer parameter types from the call arguments or from explicit type annotations in the file.

### Step 3: Extract Sink Inputs (StateFlow properties)
```
viewModel.results.collectAsState() ‚Üí property name = "results"
```
Regex: `viewModel\.(\w+)\.collectAsState\(\)`
Type inference: look for `val {propertyName} by viewModel.{propertyName}.collectAsState()` and check the import for the type, or look for type annotations.

### Output: `UIFBPSpec` data class
```kotlin
data class UIFBPSpec(
    val moduleName: String,           // "DemoUI"
    val viewModelTypeName: String,    // "DemoUIViewModel"
    val packageName: String,          // "io.codenode.demo"
    val sourceOutputs: List<PortInfo>,// [PortInfo("numA", "Double"), PortInfo("numB", "Double")]
    val sinkInputs: List<PortInfo>    // [PortInfo("results", "CalculationResults")]
)

data class PortInfo(
    val name: String,
    val typeName: String,
    val isNullable: Boolean = false
)
```

## Code Generation Design

### ViewModel Generator
Produces `{Name}ViewModel.kt` in the base package:
- Extends `androidx.lifecycle.ViewModel`
- Exposes `StateFlow` properties from State object (one per Sink input)
- Exposes emit-style method that writes Source data to State object
- Includes `reset()` method

### State Generator
Produces `{Name}State.kt` as a singleton `object`:
- `_propertyName` MutableStateFlow for each Source output and Sink input
- Public `propertyNameFlow` StateFlow for each
- `reset()` function to clear all flows

### Source CodeNode Generator
Produces `{Name}SourceCodeNode.kt` in `nodes/` subdirectory:
- `object : CodeNodeDefinition` with `category = SOURCE`
- Output ports matching ViewModel emit function parameters
- Factory method: `createContinuousGenerator` (1 out), `createSourceOut2` (2 out), `createSourceOut3` (3 out)
- Processing logic: collects from State flows and emits to graph

### Sink CodeNode Generator
Produces `{Name}SinkCodeNode.kt` in `nodes/` subdirectory:
- `object : CodeNodeDefinition` with `category = SINK`
- Input ports matching ViewModel StateFlow properties
- Factory method: `createContinuousSink` (1 in), `createSinkIn2` (2 in), `createSinkIn3` (3 in)
- Processing logic: writes received values to State flows

## File Placement

Given a UI file at: `{Module}/src/commonMain/kotlin/{package}/userInterface/{Name}.kt`

Generated files go to:
- `{Module}/src/commonMain/kotlin/{package}/{Name}ViewModel.kt` ‚Äî base package (parent of userInterface)
- `{Module}/src/commonMain/kotlin/{package}/{Name}State.kt` ‚Äî base package
- `{Module}/src/commonMain/kotlin/{package}/nodes/{Name}SourceCodeNode.kt` ‚Äî nodes subdirectory
- `{Module}/src/commonMain/kotlin/{package}/nodes/{Name}SinkCodeNode.kt` ‚Äî nodes subdirectory

## Toolbar Integration Design

Add a "Generate UI-FBP" button to TopToolbar between "Generate Module" and the divider. On click:
1. Open a file chooser filtered to `.kt` files
2. Parse the selected file with `UIComposableParser`
3. Validate the parse result (has ViewModel param, has at least one port)
4. Generate all four files via the respective generators
5. Show status message: "Generated UI-FBP interface: 4 files created for {Name}"
